1. 주제 : 사이버 보안 분야에서 GPT같은 생성형 언어 모델을 사용할 수 없음 (신뢰성, 정확성이 중요한 분야)
	-> 사이버 보안에 최적화 된 언어 모델을 만드는 것을 큰 주제로 잡음
	-> 웹사이트 뿐만 아니라 대화 플랫폼에서도 적용 가능한 자동 모니터링 기술

2. 방법론 : 마약 거래나 범죄에 사용되는 언어들은 한 의미에 대해서도 구조적인 다양성이 존재
	-> 대화나 게시물 댓글 등의 위험도를 기반으로 범죄 방지가 목표

3. Downstream Task
	3.1. (Website/Chatting group)Activity Classification -> 대 카테고리로 마약, 성범죄, 총기거래 등등을 분류
	3.2. Risk Assessment(RA) -> 해당 대화나 게시물, 댓글 등의 위험도를 분석해서 일정 위험도 이상 시에 정지, 신고
		-> 구체적인 위험도 threshold 설정이 중요할 것 같음
	3.3. Keyword Filtering -> 특정 단어에 대한 필터링으로 범죄 방지

4. (추가적으로 고려해볼 것) 계좌번호 포함 계좌이체, 거래 등에서의 추가적인 조치
